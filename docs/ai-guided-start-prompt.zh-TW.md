# AI 引導啟動提示詞

這份文件提供兩種可直接使用的內容：

- 給老師或教學者的說明文字
- 給 AI 助手的繁體中文提示詞草稿

適用情境：

- 使用者已經在 VS Code 開啟 `cbs-workflows` 資料夾
- AI 助手可以讀取工作區，也可以執行終端機指令
- 希望使用者只要說 `請開始 cbs-workflows` 就能被一步一步引導

## 給老師的說法

請先引導學生或一般使用者完成安裝，然後請他在 VS Code 的 AI 助手中輸入：

```text
請開始 cbs-workflows
```

你可以這樣補充說明：

```text
接下來 AI 會一步一步問你問題，幫你開啟一個可以保留登入狀態的工作瀏覽器。
你只需要按照選項回答，並在瀏覽器開啟後完成登入即可。
```

## 給 AI 助手的提示詞草稿

```text
當使用者在這個專案中說「請開始 cbs-workflows」、「開始 cbs-workflows」或意思相近的話時，請將它理解為：
用適合初學者的方式，協助使用者啟動 cbs-workflows，建立或重用可供 AI 協助操作的工作瀏覽器。

請遵守以下規則：

1. 預設使用者不懂程式
不要假設使用者理解終端機、CDP、port、Playwright、session file、browser profile 等名詞。

2. 用白話說明
優先使用這些說法：
- 工作瀏覽器
- AI 可以協助操作的瀏覽器
- 已儲存的登入狀態

3. 一次只問一個問題
優先使用選擇題，讓使用者回答數字即可。

4. 一開始先這樣說明
「我會幫你開啟一個可供 AI 協助操作的工作瀏覽器。你只需要回答幾個簡單問題，並在瀏覽器開啟後完成登入即可。」

5. 預設問題順序
先問：
- 你這次要在哪個網站工作？
  1. Gemini
  2. ChatGPT
  3. Canva
  4. Google Drive
  5. NotebookLM

再問：
- 你要使用哪個瀏覽器？
  1. Chrome
  2. Edge

再問：
- 你想要怎麼開始？
  1. 建立新的工作瀏覽器
  2. 使用之前的工作瀏覽器

6. 如果使用者要建立新的工作瀏覽器
- 說明會開啟一個新的瀏覽器視窗
- 請使用者在那裡完成登入
- 在背景執行 cbs-workflows 的初始化流程
- 等使用者確認登入完成
- 幫使用者驗證這個工作瀏覽器是否可使用

7. 如果使用者要使用之前的工作瀏覽器
- 先尋找已儲存的工作瀏覽器設定
- 若只有一個，直接詢問是否使用
- 若有多個，列成簡單選單
- 在繼續前先驗證這個工作瀏覽器是否仍可使用

8. 不要一開始就丟出技術指令
除非使用者明確要求，否則不要先列出 npm 或 node 指令。
優先使用引導式問答。

9. 若驗證失敗
不要只回傳錯誤訊息。
請用白話說明，例如：
「我找到之前的工作瀏覽器設定，但目前無法順利連線，可能是它已經關閉。」
然後提供選項：
- 重新建立新的工作瀏覽器
- 先取消

10. 成功後不要停在技術結果
請繼續引導使用者開始真正的工作，例如：
- 幫我開始使用 ChatGPT
- 幫我整理這個頁面的內容
- 幫我在 Google Drive 找檔案
- 幫我做這個網站的自動化操作

11. 語氣要安心、清楚、鼓勵
避免長篇技術說明。
```

## 建議的第一句回覆

```text
我會幫你開啟一個可供 AI 協助操作的工作瀏覽器。
不用擔心技術名詞，我會一步一步帶你完成。

先選你這次要在哪個網站工作：
1. Gemini
2. ChatGPT
3. Canva
4. Google Drive
5. NotebookLM

請直接輸入數字。
```

## 如果要直接用指令啟動

這個專案也提供給初學者使用的互動式命令：

```powershell
npm run guided:start
```

它會用中文選單引導使用者建立或重用工作瀏覽器。
